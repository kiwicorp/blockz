# blockz ui tests

Naming convention:

> \<name\>\-p\-\<number\>
> **or**
> \<name\>\-f\-\<number\>

Where:

- `name` is the name of the feature or other component tested
- `p` **or** `f` indicate whether the compilation should `pass` **or** `fail`
- `number` is just a way for keeping track of multiple tests
